# 首页精选专题数量入口设计

## 目标

让用户在首页“GitHub 明星项目”区域明确知道每个精选专题包含多个项目，并能直接进入专题页浏览全部项目。

## 当前问题

首页每个精选专题只展示一个 Top pick。虽然专题标题和卡片可以进入专题页，但界面没有明确显示专题内还有其他项目，用户容易误以为每个专题只有一个推荐项。

## 方案

保留现有专题标题、说明和 Top pick 卡片。在每个专题的 Top pick 下方增加独立入口：

```text
查看全部 N 个项目 →
```

- `N` 使用该专题 API 数据中的 `repositories.length`。
- 入口链接到现有专题页 `/collections/{slug}`。
- 不在首页展开更多项目卡片，避免右侧栏过长和信息密度失控。
- 入口沿用当前页面的 cyan 强调色和链接交互样式。
- 专题没有项目时不渲染 Top pick 和数量入口，沿用当前空数据处理。

## 数据流

后端 `/api/featured` 已返回每个专题的完整 `repositories` 数组，前端无需新增接口或修改数据结构。首页读取数组长度生成数量文案，并使用专题 `slug` 构造链接。

## 测试

前端测试需要验证：

- 首页为包含多个项目的专题渲染正确的“查看全部 N 个项目”文案。
- 数量入口链接到对应的 `/collections/{slug}`。
- 现有 Top pick 展示保持不变。

## 不在范围内

- 不修改精选生成数量和评分规则。
- 不修改专题页布局。
- 不在首页展示第二、第三个项目卡片。
- 不修改后端 API。
